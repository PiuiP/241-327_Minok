# Конспект к лабораторной работе №1
## Трёхуровневая клиент-серверная архитектура

### Определения

**REST** (Representational State Transfer/Передача представительского статуса) — архитектурный стиль построения API, основанный на HTTP-методах (GET, POST, PUT, PATCH, DELETE) и использовании URL для доступа к ресурсам.

**Приложение Django** — модуль, выполняющий определенную функцию в проекте (например, mainapp). Создается командой `startapp`.

**ASGI** (Asynchronous Server Gateway Interface/Интерфейс шлюза асинхронного сервера) — интерфейс для асинхронных веб-серверов и приложений.

**WSGI** (Web Server Gateway Interface/Интерфейс шлюза веб-сервера) — стандарт синхронного взаимодействия веб-сервера и приложений на Python.

**URL** (Uniform Resource Locator/Единый указатель ресурсов) — адрес ресурса в интернете.

**URI** (Uniform Resource Identifier/Единый идентификатор ресурса) — идентификатор ресурса (включает URL и URN).

**UTC** (Coordinated Universal Time) — всемирное координированное время, стандарт времени.

**Транзакция** — группа операций с БД, которые выполняются как единое целое (либо все успешно, либо откат).

**Миграции баз данных** — способ изменения схемы БД с сохранением данных. В Django создаются командой `makemigrations` и применяются `migrate`.

### Структура Django-проекта

| Файл | Назначение |
|------|------------|
| `manage.py` | Главный скрипт управления проектом (запуск сервера, миграции, создание приложений) |
| `settings.py` | Настройки проекта (БД, установленные приложения, middleware, пути) |
| `urls.py` | Маршрутизация URL — связывает URL с обработчиками (views) |
| `models.py` | Описание моделей данных (таблиц в БД) |
| `views.py` | Обработчики HTTP-запросов (контроллеры) |
| `serializers.py` | Преобразование моделей в JSON и обратно (создается вручную) |

### Современные фреймворки для бекенда (2025)

1. **FastAPI** — асинхронный, быстрый, с автоматической генерацией OpenAPI
2. **Django + DRF** — монолитный, "все включено", админка, ORM
3. **Litestar** —  ASGI-фреймворк, альтернатива FastAPI и Django, более высокая производительность, лучше модульность

**Почему лабораторная на Django?** — Встроенная админка, мощная ORM(Object-Relational Mapping), миграции, большое сообщество, подходит для учебных целей.

### SQLite (бессерверная БД)

**Ниши применения:**
- Разработка и тестирование
- Мобильные приложения
- Небольшие проекты (до 1 млн записей)

**Преимущества:**
- Не требует установки и настройки сервера
- Один файл — вся БД
- Легковесная

**Недостатки:**
- Нет многопользовательского доступа для записи
- Ограниченная масштабируемость
- Меньше типов данных

### ORM (Object-Relational Mapping)

**Расшифровка:** Object-Relational Mapping

**Определение:** Технология связывания объектов языка программирования с таблицами реляционной БД.

**Библиотеки ORM для Python:**
- Django ORM (встроенная)
- SQLAlchemy
- Peewee
- SQLModel

**Принцип работы:** Класс на Python → таблица в SQL, экземпляр класса → строка таблицы, атрибуты → колонки.

**Преимущества ORM:**
- Не нужно писать сырой SQL
- Защита от SQL-инъекций
- Унификация кода
- Легкая смена БД

**Недостатки ORM:**
- Сложные запросы могут быть неэффективными
- Дополнительная нагрузка на CPU
- Изучение специфики ORM

**Ниши применения ORM:**
- CRUD-приложения (типовые операции)
- Проекты с частым изменением схемы БД

**Альтернативы ORM для миграций:**
- Ручные SQL-скрипты (ALTER TABLE, CREATE TABLE)
- Инструменты типа Alembic (для SQLAlchemy), Flyway, Liquibase

### Фреймворки и языки для высоконагруженных проектов в 2025 году

- **Go** -  **Gin/Fiber/Echo** -  высокая производтельность, быстрая компиляция, легковесные горутины вместо потоков, отличная работа с конкурентностью
- **Rust** - **Actix Web/Axum** - максимальная производительность (на уровне C++), безопасная работа с памятью без GC, предсказуемая задержка, отсутствие сборщика мусора
- **Python** - **FastAPI/Litestar (async)** - достаточно для многих highload-сценариев (до 10-20k req/sec), быстрая разработка, огромная экосистема библиотек
- **Java** - **Spring Boot/Quarkus** - зрелая экосистема, проверенная временем, JIT-компиляция, многопоточность из коробки, огромное сообщество
- **Kotlin** - **Ktor/Spring Boot** - современный синтаксис, корутины (как горутины в Go), полная совместимость с Java-экосистемой
- **C#** - **ASP.NET Core** - высокая производительность (TechEmpower benchmarks), отличная асинхронность, развитая экосистема от Microsoft
- **Node.js (TypeScript)** - **NestJS/Fastify** - событийно-ориентированная архитектура, хорош для I/O-bound задач, единый язык для фронта и бэка

### Хранение паролей

Пароли должны храниться только в **захэшированном** виде (bcrypt, PBKDF2, Argon2). Никогда в открытом виде или с обратимым шифрованием.

### Пользователь БД vs суперпользователь

- **Обычный пользователь (django)** — минимальные привилегии (SELECT, INSERT, UPDATE, DELETE для таблиц приложения)
- **Суперпользователь (postgres)** — полный доступ ко всем БД, создание/удаление пользователей

**Зачем отдельный пользователь?** Принцип минимальных привилегий — безопасность.

### Процесс vs Поток (thread)

- **Процесс** — экземпляр программы в памяти, изолированное адресное пространство
- **Поток** — единица выполнения внутри процесса, разделяет память процесса

**Области применения потоков:** I/O-операции, конкурентная обработка запросов, асинхронные задачи.
